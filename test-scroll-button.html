スクロール復元時のトップボタン表示確認手順

注意： ブラウザの開発者ツールを開いてコンソールログを確認してください。

動作確認手順

  1. 開発サーバーにアクセス
    http://localhost:3000 を開く
  2. コンソールログの確認
    F12キーで開発者ツールを開き、Consoleタブを選択
  3. 記事リストをスクロール
    記事リストを下にスクロールして、複数ページを読み込む
    （右下にトップボタンが表示されることを確認）
  4. 記事詳細ページへ移動
    任意の記事をクリックして詳細ページへ移動
  5. 一覧ページに戻る
    ブラウザの戻るボタンまたは「戻る」リンクをクリック
  6. コンソールログを確認
    以下のログが出力されているか確認：
    • [ScrollToTopButton] イベントリスナー登録
    • [ScrollRestoration] 復元完了イベント発火 - scrollY: [数値]
    • [ScrollToTopButton] イベント受信 - scrollY: [数値] restored: true cancelled: false
    • [ScrollToTopButton] スクロール位置再チェック - currentScrollY: [数値]
  7. トップボタンの表示確認
    スクロール復元後、約1秒待ってトップボタンが表示されるか確認

問題が発生した場合

  • イベントが発火していない → useScrollRestorationの処理を確認
  • イベントは発火しているが受信されていない → ScrollToTopButtonのマウントタイミングを確認
  • スクロール位置が0になっている → スムーススクロールのタイミング問題
